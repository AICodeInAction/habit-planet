# 习惯星球 - 产品设计文档

## 1. 产品定位

**习惯星球**是一款面向6-12岁儿童及其家庭的习惯养成应用，通过星际探索的游戏化设计，帮助孩子在有趣的互动中培养良好生活习惯，同时增强家庭成员之间的互动和鼓励。

## 2. 用户体验分析

### 2.1 用户画像

**主要用户 - 儿童（6-12岁）**
- **需求**：有趣的互动、即时反馈、成就感、认可和鼓励
- **行为特征**：注意力集中时间短、喜欢视觉刺激、需要持续激励
- **痛点**：传统习惯养成枯燥乏味、缺乏动力、容易放弃

**次要用户 - 家长**
- **需求**：监督孩子习惯养成、亲子互动、数据追踪
- **行为特征**：关注孩子成长、希望参与但不干涉、需要简单易用的工具
- **痛点**：难以持续监督、缺乏有效的激励方法、亲子互动不足

### 2.2 核心用户场景

#### 场景1：早晨打卡
- **时间**：7:00-8:00
- **地点**：家中
- **流程**：
  1. 完成刷牙洗脸
  2. 打开应用，看到今日习惯列表
  3. 一键打卡"刷牙洗脸"
  4. 获得即时反馈：星星+1、连续天数+1
  5. 看到星际地图进度条前进

#### 场景2：睡前回顾
- **时间**：21:00-21:30
- **地点**：家中
- **流程**：
  1. 和父母一起查看今日完成情况
  2. 父母为孩子的成就点赞
  3. 孩子看到解锁了新的装饰物
  4. 设定明天的目标
  5. 满足感入睡

#### 场景3：周末探索
- **时间**：周末上午
- **地点**：家中
- **流程**：
  1. 查看本周数据统计
  2. 发现解锁了新星球
  3. 装饰新星球
  4. 查看家庭排行
  5. 与家人分享成就

### 3.2 核心功能详解

#### 🌌 星际地图系统
**设计目的**：可视化进度，提供长期目标感

**功能特点**：
- 星球作为阶段性目标
- 通过星星解锁新星球
- 每个星球有独特主题和装饰
- 进度动画反馈
- 当前位置清晰标识

**交互流程**：
1. 用户进入应用，首先看到星际地图
2. 当前正在探索的星球居中显示
3. 已解锁星球显示完成标识
4. 未解锁星球显示锁定状态和解锁条件
5. 点击星球可查看详情和装饰

#### ✅ 习惯打卡系统
**设计目的**：简化打卡流程，降低操作门槛

**功能特点**：
- 按时段分组，符合儿童作息
- 一键打卡，操作简单
- 连续天数显示，强化激励
- 今日进度一目了然
- 日历回顾历史记录

**打卡机制**：
- 每个习惯每天可打卡一次
- 打卡后获得2颗星星
- 连续打卡增加连击数
- 完成当天所有习惯获得额外奖励
- 支持补打卡（24小时内）

#### 🏆 成就系统
**设计目的**：提供持续激励，增强成就感

**奖牌类型**：

1. **传奇奖牌**（基于连续性）
   - 习惯大师：连续7天
   - 钻石习惯：连续30天
   - 习惯之王：连续100天

2. **星星收集者**（基于积分）
   - 星星新手：50星星
   - 星光闪耀：100星星
   - 璀璨星河：500星星

3. **星际探险家**（基于探索）
   - 地球守护者：解锁地球
   - 水之使者：解锁水星球
   - 火焰之心：探索火星球

4. **家庭荣耀**（基于互动）
   - 温馨家庭：家人互动10次
   - 互助之星：获得50个赞
   - 超级家庭：家庭总星星1000

#### 👨‍👩‍👧‍👦 家庭互动系统
**设计目的**：增强亲子互动，提供社交激励

**核心功能**：
- 家庭成员数据展示和对比
- 相互点赞鼓励机制
- 家庭动态时间线
- 家庭挑战任务
- 家庭总成就统计

**互动机制**：
- 家长可以为孩子的成就点赞
- 点赞增加孩子的成就感
- 家庭成员互相可见进度
- 共同完成家庭挑战
- 家庭排行榜激励

## 4. UI/UX 设计原则

### 4.1 视觉设计

#### 色彩系统
**主色调**：
- 主色：#5b67f8（科技蓝）- 代表探索和未来
- 辅色：#764ba2（神秘紫）- 代表星际和梦想
- 成功色：#4ecdc4（清新绿）- 代表完成和成就
- 警示色：#ff6b6b（温暖红）- 代表热情和鼓励

**渐变使用**：
- 卡片背景使用柔和渐变
- 按钮使用有方向感的渐变
- 增强视觉层次和吸引力

#### 图标系统
- 使用 FontAwesome 6.5.1
- 配合 Emoji 增强趣味性
- 图标大小保持一致性
- 颜色与功能关联

#### 间距系统
- 基础单位：4px
- 小间距：8px
- 中间距：16px
- 大间距：24px
- 超大间距：32px

### 4.2 交互设计

#### 反馈机制
1. **即时反馈**
   - 打卡后立即显示动画
   - 星星飞入效果
   - 进度条动画
   - 震动反馈（移动端）

2. **延迟反馈**
   - 达成成就时的弹窗
   - 解锁新星球的动画
   - 每日/每周总结

3. **社交反馈**
   - 家人点赞通知
   - 家庭成员完成通知
   - 家庭挑战进度

#### 动画设计
- **加载动画**：星球旋转、星星闪烁
- **转场动画**：渐入渐出、滑动切换
- **反馈动画**：缩放、抖动、飞入
- **装饰动画**：悬浮、闪烁、旋转

### 4.3 适配iOS设计规范

#### 状态栏
- 高度：44px
- 显示时间、信号、电量
- 颜色根据背景自适应

#### 导航栏
- 底部 Tab Bar
- 高度：85px（包含安全区域）
- 5个主要入口
- 当前页面高亮显示
- 毛玻璃效果背景

#### 卡片设计
- 圆角：16px
- 阴影：0 2px 8px rgba(0,0,0,0.06)
- 白色背景
- 内边距：16px

#### 按钮设计
- 主要按钮：渐变背景，白色文字
- 次要按钮：边框样式
- 圆角：20px
- 高度：44px（符合苹果规范）

## 5. 游戏化设计

### 5.1 核心循环

```
打卡习惯 → 获得星星 → 解锁星球 → 获得装饰物 → 展示成就 → 持续打卡
```

### 5.2 激励系统

#### 短期激励（每日）
- 打卡即时反馈
- 星星积分
- 进度条增长
- 今日完成统计

#### 中期激励（每周）
- 连续打卡天数
- 周报统计
- 解锁装饰物
- 家庭排名

#### 长期激励（长期）
- 星球解锁
- 成就奖牌
- 等级提升
- 探索进度

### 5.3 成长系统

**等级体系**：
- Level 1-5：新手探险家（0-100星星）
- Level 6-10：初级航行者（101-300星星）
- Level 11-15：资深旅行家（301-600星星）
- Level 16-20：传奇探险家（601-1000星星）
- Level 20+：宇宙主宰（1000+星星）

**星球体系**：
1. 地球（新手星球）- 免费解锁
2. 水之星球 - 50星星解锁
3. 火焰星球 - 100星星解锁
4. 森林星球 - 200星星解锁
5. 冰雪星球 - 500星星解锁
6. 梦幻星球 - 1000星星解锁

## 7. 技术实现建议

### 7.1 前端架构

**技术选型**：
- React/React Native（跨平台）
- Redux（状态管理）
- React Navigation（路由）
- Animated API（动画）

**核心模块**：
```
src/
├── components/          # 通用组件
│   ├── Button/
│   ├── Card/
│   ├── Avatar/
│   └── ...
├── screens/            # 页面
│   ├── Home/
│   ├── Habits/
│   ├── Achievements/
│   ├── Family/
│   └── Profile/
├── navigation/         # 导航配置
├── store/             # 状态管理
│   ├── habits/
│   ├── user/
│   ├── family/
│   └── achievements/
├── services/          # API服务
├── utils/            # 工具函数
└── assets/           # 静态资源
```

### 7.2 后端架构

**技术选型**：
- Node.js + Express（或Python + FastAPI）
- PostgreSQL（关系型数据）
- Redis（缓存）
- AWS S3（图片存储）

**核心API**：
```
POST   /api/auth/login              # 登录
POST   /api/auth/register           # 注册
GET    /api/user/profile            # 用户信息
GET    /api/habits                  # 获取习惯列表
POST   /api/habits/check           # 打卡
GET    /api/achievements           # 获取成就
GET    /api/family/members         # 家庭成员
POST   /api/family/like            # 点赞
GET    /api/stats/weekly           # 周报统计
```

### 7.3 数据模型

**用户表（users）**：
```sql
id, username, email, password_hash, avatar_url, 
level, stars, created_at, updated_at
```

**习惯表（habits）**：
```sql
id, user_id, name, icon, color, time_period,
target_days, created_at, updated_at
```

**打卡记录表（check_ins）**：
```sql
id, user_id, habit_id, check_date, stars_earned,
created_at
```

**成就表（achievements）**：
```sql
id, user_id, achievement_type, achievement_name,
unlocked_at
```

**家庭表（families）**：
```sql
id, family_name, created_at, updated_at
```

**家庭成员关系表（family_members）**：
```sql
id, family_id, user_id, role, joined_at
```
